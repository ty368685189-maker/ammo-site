这个文件给宇少和后续 Codex 会话快速判断：现在项目做到哪一步，哪些内容可以入库，哪些需要暂缓。

## 1. 当前主线

当前主线：先做厚本地入库预备级知识库，正式入库后置；入库时仍然数量优先、质量过闸、保守入库。

- 知识库就是企业和院校入库预备信息池，位置是 `research-drafts/knowledge-pool/`。
- Codex 和 Antigravity 扩知识库必须使用同一套规则：`research-drafts/knowledge-pool/rules/SHARED_CODEX_ANTIGRAVITY_RULES.md`。
- 当前先慢慢把企业、院校、研究所、检测评价机构、爆破工程公司、安全/含能/兵器/矿业安全相关院校入口攒起来。
- 知识库资料要尽量达到后续可入库条件：名称、地区、方向、关键链接、来源记录、证据强弱、重复风险、建议动作、检索记录和入库预备字段都要尽量补齐。
- Grok 暂停作为主流程，因为搜索和整理不稳定；后续优先由 Codex 或 Antigravity 做知识库。
- 企业和院校正式库数量后续再慢慢从知识库里挑强候选清洗入库。
- clean 草稿只要 `dry-run` 通过、评分 85 分及以上、没有阻塞项，就可以正式入库。
- 宇少暂时不做人工逐条核对，人工核对后置。
- 入库时必须保守：官网、招生网、招聘入口等链接要尽量给；薪资、作息、分数线、就业率、招生人数等没强证据就写 `未知`、`待核对` 或放进风险。
- 不在弱字段上耗太多时间；但不能硬编，不能把弱证据写成确定事实。

原因：宇少当前更在乎企业/院校覆盖量。这个网站先作为学生查入口和做初筛的导航站，不追求每个字段都达到人工审稿级别。

## 3. 草稿区状态

草稿区目录：

- 知识库候选池：`research-drafts/knowledge-pool/`
- 企业草稿：`research-drafts/companies/`
- 院校草稿：`research-drafts/universities/`

知识库候选池的定位：

- 先搜集企业和院校信息，不直接显示到网站。
- 可以由 Codex 或 Antigravity 继续做。
- 根目录只放入口和规则；批次 Markdown 放 `research-drafts/knowledge-pool/batches/`，raw JSON 放 `research-drafts/knowledge-pool/raw-json/`。
- 每条按入库预备级标准保留名称、类型、地区、相关方向、关键入口链接、来源记录、证据强弱、重复风险、建议动作、检索记录和待核对字段。
- 不需要跑导入脚本，不需要 build，不改正式库。
- 后续正式入库时，再从知识库里挑强候选做 clean JSON。

这些 clean 草稿的定位：

- `dry-run` 通过、评分 85 分及以上、无阻塞项：可以正式导入。
- 70-84 分：先修明显问题，再考虑入库。
- 0-69 分或有阻塞项：暂缓，不入库。
- 人工打开来源逐条核对可以后置。
- 入库后仍要保留 `待核对`、`未知`、风险说明，不能写成完全确定。

企业资料会话和院校资料会话已暂停，后续扩库与入库统一由本总控会话执行，避免频繁弹出 Command approval。

当前执行节奏：

- 企业线优先扩量，每批优先导入 3-5 家评分达标企业。
- 院校线谨慎补强，每批优先导入 2-4 所官方本科证据明确的院校。
- 企业线可以继续从 raw 候选里清洗；院校线不建议硬扩普通安全工程或纯科研相关院校。
- 后续不再把任务派给旧企业/院校会话，减少手动 approve 打扰。

企业扩库不再盲目追加，按结构补齐：

- 地域：优先补华东、华南、西北、西南等区域空白或偏少省份；避免河南、四川、陕西等已较多区域继续扎堆。
- 城市层级：兼顾一线/省会、产业城市、资源型城市和县域生产基地；列表中保留城市与区域，方便学生判断工作地点。
- 企业性质：当前国企较多，后续适当补民企、上市民爆企业、地方平台企业和专业爆破服务企业。
- 业务形态：保持民爆产品、火工品/雷管、爆破工程、含能材料、智能装备、安全技术等类型均衡。
- 入口优先级：有官网、集团页、招聘公告、学校就业网或政府/上市公司公告的优先；只有百科/工商聚合页的暂缓。
- 重复控制：旧名、重组前主体、子公司与已入库总部高度重合时，不重复入库，除非能提供独立招聘或独立业务入口。

## 6. 下一步建议

建议路线：企业已到 50 家，院校已到 32 所，batch10 / batch11 链接审计已完成。下一步先继续做厚知识库候选池，正式入库慢慢推进。

顺序：

3. 知识库阶段不深挖待遇、作息、分数线、招生人数、就业率；没有强证据就写 `未知` / `待核对`。
4. 企业知识库继续补东北、华南、华北、西北入口；民企、地方平台企业、专业爆破服务、安全检测评价、电子雷管装备、含能材料类优先。
5. 院校知识库可以收本科强相关、考研相关、跨考参考，但必须标清口径，不把研究生方向包装成本科专业。
6. 正式入库时，从知识库挑强候选，每批企业 5-8 家或院校 2-4 所，生成 clean JSON。
7. 每份 clean 继续跑 dry-run 和 score，85 分及以上且无阻塞项才正式导入。
8. 每批入库后运行 `npm run build`、`npm run check:data-quality`，必要时跑 `npm run qa:routes`。

部署建议：

- 项目是纯前端静态站。
- 构建产物在 `dist/`。
- 对宇少更友好的方式是 Vercel、Netlify、Cloudflare Pages 这类静态托管。
- 不建议优先折腾 VPS 和 Nginx，除非后续明确需要。
