# 弹药工程与爆炸技术导航站 — 项目规格

## 项目定位
"给弹药孩子一个温暖的家" — 弹药工程与爆炸技术专业的一站式导航网站。
目标用户：在读生、考研党、毕业生、想了解这个专业的人。

核心原则：**用户打开网站，3秒内能找到他想要的信息。所有内容都是问答式/清单式，不写散文，不写宣传稿。**

## 技术栈
- 前端：Vue 3 + Vite + TailwindCSS
- 后台管理：Directus（Docker 部署，用户通过可视化后台填数据）
- 部署：Nginx（OpenResty），域名待定（可能是 ammo.yugold.top）

## 页面结构（共6个页面）

### 1. 首页 (/)
- 大标题："给弹药孩子一个温暖的家"
- 副标题："弹药工程与爆炸技术专业一站式导航"
- 5个板块入口卡片（专业介绍、企业名录、院校一览、考研方向、经验分享）
- 网站简介（一段话，3-5行）
- 底部导航：关于、投稿、联系方式

### 2. 专业介绍 (/about-major)
纯内容页，分区块展示：
- 这个专业学什么课（按大一到大四分阶段）
- 日常是什么样的（实验室、课表、男女比例等实际情况）
- 毕业能干什么工作（具体岗位，不是"广阔前景"）
- 工资大概多少（按不同去向给出区间）
- 这个专业危险吗（实际回答这个问题）

### 3. 企业名录 (/companies)
- 顶部筛选栏：按类型筛选（军工/民爆/安防/科研）、按城市筛选、搜索框
- 企业卡片列表，每个卡片显示：名称、类型标签、城市、薪资范围、学历要求
- 点击进入企业详情页

企业详情页（/companies/:id）：
- 企业名称、类型、所在城市
- 招不招这个专业（直接说招不招、招什么岗位）
- 学历要求（本科/硕士/博士）
- 大概薪资（应届生到手多少）
- 加班情况（996？正常双休？经常出差？）
- 有没有编制（国企编制、合同制、劳务派遣）
- 官网/招聘链接

### 4. 院校一览 (/universities)
- 顶部筛选栏：按层次筛选（985/211/双一流/普通本科）、搜索框
- 院校卡片列表，每个卡片显示：校名、城市、层次标签、专业排名
- 点击进入院校详情页

院校详情页（/universities/:id）：
- 学校名称、城市、层次
- 这个专业排名（学科评估等级）
- 录取分数线（近三年，可展开看详情）
- 招多少人（每年招生计划）
- 毕业去向（这个学校毕业的人都去哪了）
- 一句话评价（性价比、地理位置、行业资源）

### 5. 考研方向 (/graduate)
- 考研方向列表，每个方向可展开
- 每个方向显示：方向名称、考试科目、哪些学校招、难度、一句话建议
- 经验分享区（帖子列表，按时间排序）
- 经验分享详情页（/graduate/:id）

### 6. 关于我们 (/about)
- 建站初衷
- 投稿方式（告诉用户怎么提交内容）
- 免责声明（信息仅供参考，数据来自公开渠道）

## 视觉风格

### 配色
- 主色：深蓝/藏青 #1a1a2e（背景）
- 强调色：暖橙 #e94560（按钮、链接、高亮）
- 辅助色：浅灰 #f5f5f5（卡片背景，暗色模式下用 #2a2a3e）
- 文字：白色 #ffffff / 浅灰 #cccccc

### 风格关键词
- 军工科技感 + 温暖社区感
- 圆角卡片、微阴影、hover 发光效果
- 深色主题为主，可选浅色模式
- 参考：Vercel 官网的科技感 + 少数派的社区感

### 响应式
- PC 优先，但必须适配手机
- 移动端：单列布局，卡片全宽
- PC端：卡片网格布局（2-3列）

## 数据来源
前端通过 Directus API 获取数据，用户通过 Directus 后台管理界面录入/编辑数据。
所有数据表结构见 DATA_SCHEMA.md。
