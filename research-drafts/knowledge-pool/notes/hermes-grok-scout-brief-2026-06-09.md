## 2. Hermes 的角色

Hermes / Grok 适合做：

- 扩大企业名单。
- 搜索企业名称、城市、省份、相关方向。
- 找官网、集团页、招聘入口、学校就业网、政府资料、公告等公开链接。
- 记录实际检索关键词。
- 标注“可能相关线索”和“不确定点”。

Hermes / Grok 的成果名称统一叫：

```text
企业基础线索包
```

意思是：只给 Codex / Antigravity 减少“找企业名字”的负担，不代表已经完成知识库搜集。

Hermes 不需要精确知道本地正式库和知识库已经有哪些企业。重复是正常现象，不算 Hermes 任务失败。

后续精确去重由 Codex / Antigravity 完成。

Hermes / Grok 不负责：

- 不判断“确定可以入库”。
- 不产出合格知识库成品。
- 不替代 Codex / Antigravity 的深度搜索。
- 不写正式库。

## 3. 搜索必须有批次主题

Hermes 不要无规则地乱搜。每一批必须先选一个清楚的搜索主题。

推荐每批 10 条。每批只选下面一种模式，最多同时叠加 2 个条件。

### 模式 A：按行业方向搜

适合补某条产业链。

示例主题：

- 电子雷管芯片 / 控制模块 / 起爆器。
- 地方民爆器材 / 工业炸药 / 雷管生产。
- 爆破工程 / 矿山爆破 / 隧道爆破。
- 防爆检测 / 爆破振动监测 / 爆炸安全评价。
- 含能材料 / 火工品 / 引信 / 推进剂 / 装药。
- 智能民爆装备 / 电子雷管生产线 / 自动化装配。

### 模式 B：按区域或城市搜

适合补正式库薄弱地区。

示例主题：

- 东北地区民爆和爆破工程企业。
- 华南地区防爆检测和安全评价企业。
- 西北地区民爆生产和军工配套企业。
- 云南 / 贵州 / 广西地方民爆企业。
- 江苏 / 浙江 / 上海电子雷管上游企业。
- 陕西 / 山西 / 河南火工品和民爆企业。

### 模式 C：按企业性质搜

适合区分求职体验和就业方向。

示例主题：

- 国企 / 地方国资民爆企业。
- 民营爆破工程公司。
- 上市公司上游供应商。
- 科研院所 / 军工研究所。
- 检测机构 / 认证机构 / 安全评价机构。
- 专精特新 / 小巨人企业。

### 模式 D：按规模和入口质量搜

适合找后续更容易核验的对象。

示例主题：

- 有独立官网的企业。
- 有校园招聘或学校就业网记录的企业。
- 有政府公示、资质公告或上市公告的企业。
- 非头部但有公开招聘入口的地方企业。

### 模式 E：按产业链位置搜

适合补全专业相关就业面。

示例主题：

- 上游：芯片、控制模块、敏感元件、原材料。
- 中游：工业炸药、电子雷管、起爆器材、火工品制造。
- 下游：爆破工程、矿山服务、隧道爆破、安全监测。
- 支撑：防爆检测、安全评价、爆炸防护、智能装备。

规则：

- 弱来源只能当线索，不能当强证据。

- 没找到独立招聘页时，招聘入口写“待核对”，不要用官网首页冒充招聘入口。

补充规则：

- moomoo、东方财富、雪球、同花顺这类财经页面，不要写成“上市公司公告”，写“第三方财经资料”。

## 10. 对接 Codex / Antigravity 的交接要求

Hermes 的输出要让 Codex / Antigravity 能直接接着干。

每条企业不要只写“看起来相关”，还要写清楚后续该怎么处理。

### 每条必须有一个处理状态

```text
后续处理建议：优先深查 / 普通深查 / 暂缓 / 仅作线索 / 疑似重复先核对
```

含义：

- 优先深查：官网、招聘、方向都比较清楚，值得 Codex / Antigravity 先核验。
- 普通深查：方向可能相关，但还缺招聘、官网或官方来源。
- 暂缓：证据薄、主体不清、方向偏远。
- 仅作线索：只有名字或第三方页面，暂不适合进入知识库。
- 疑似重复先核对：可能和正式库、知识库、集团主体、子公司、旧名重组重复。

### 每条必须写清楚“下一步查什么”

不要只写“待核对”，要具体到动作：

```text
建议 Codex / Antigravity 下一步核验：
- 查官网是否有具体产品/业务页。
- 查是否有校园招聘或学校就业网原始招聘简章。
- 查是否和某某集团、某某已入库主体存在母子公司或重组关系。
- 查方向词是否有官方来源支撑。
```

### 每条必须给后续检索关键词

除了 Hermes 已经搜过的词，还要给 2-4 个推荐给 Codex / Antigravity 的深查关键词。

示例：

```text
建议深查关键词：
- 企业全称 官网 产品 电子雷管
- 企业全称 校园招聘 弹药工程
- 企业全称 学校就业网
- 企业全称 政府公示 安全评价资质
```

### 每条必须把来源角色写清楚

来源不要只列链接，还要写它的用途：

```text
来源角色：主入口 / 业务证明 / 招聘线索 / 资质线索 / 重复关系线索 / 弱背景线索
```

这样 Codex / Antigravity 后续能知道哪个链接该重点打开。

### 每条必须写清主体关系

如果出现集团、子公司、旧名、简称，要写：

```text
主体关系线索：
- 母公司 / 集团：待核对
- 子公司 / 分支机构：待核对
- 曾用名 / 简称：待核对
- 可能混淆对象：待核对
```

不确定就写“待核对”，不要空着。
