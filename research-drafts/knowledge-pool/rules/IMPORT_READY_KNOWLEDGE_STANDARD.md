# 入库预备级知识库标准

这个文件定义“知识库资料要做到什么程度”。

宇少当前目标不是随便攒链接，而是在本地扩大一个“入库预备库”：资料质量尽量接近正式入库要求，但暂时不写正式库、不改页面、不部署。

## 1. 核心原则

知识库 = 企业和院校的入库预备资料库。

每条资料要做到：

- 后续可以较快转换成 clean JSON。
- 来源链接可追溯。
- 证据强弱写清楚。
- 风险和待核对字段写清楚。
- 不把待核对内容写成确定事实。

但知识库阶段不要：

- 不改 `src/data/companies.ts`。
- 不改 `src/data/universities.ts`。
- 不运行正式导入。
- 不改页面。
- 不部署。
- 不碰服务器、VPS、域名。

## 2. 企业入库预备字段

企业候选至少准备这些字段：

- 名称。
- 类型：企业。
- 所在城市。
- 所在省份 / 区域。
- 企业性质：国企 / 民企 / 外企 / 创业公司 / 待核对。
- 相关方向：民爆产品 / 电子雷管 / 爆破工程 / 含能材料 / 智能装备 / 安全检测评价 / 军工科研院所等。
- 官网或主要入口。
- 招聘入口或招聘公告，没有就写待核对。
- 关键链接列表。
- 来源记录：标题、类型、日期、URL。
- 常见岗位线索：没有强证据写待核对。
- 学历要求线索：没有强证据写待核对。
- 薪资：没有强/中证据写未知。
- 作息：没有强/中证据写待核对。
- 简短描述：40 字以内，客观，不宣传。
- 适合人群：2-4 条。
- 风险和待核对：2-5 条。
- 检索关键词和检索轮次。
- 重复风险：正式库重复、知识库重复、集团/子公司重叠、旧名/重组前主体。
- 建议动作：后续清洗 / 暂缓 / 跳过。

企业强候选建议至少 3 个来源，其中最好有：

- 官网。
- 集团页。
- 招聘公告。
- 学校就业网。
- 政府资料。
- 上市公司公告。
- 可靠招聘入口。

## 3. 院校入库预备字段

院校候选至少准备这些字段：

- 学校名称。
- 类型：院校。
- 城市。
- 省份。
- 层次：985 / 211、211、普通本科，无法确认写待核对。
- 候选分层：本科强相关 / 本科近相关 / 考研相关 / 跨考参考 / 暂缓。
- 本科/考研/跨考口径。
- 相关方向：弹药工程与爆炸技术、兵器科学与技术、含能材料、爆炸力学、安全科学与工程、矿业安全等。
- 学校官网。
- 招生网。
- 学院官网。
- 本科专业入口或研究生招生目录入口。
- 关键链接列表。
- 来源记录：标题、类型、日期、URL。
- 专业/方向线索：必须标清本科、研究生或跨考。
- 简短概览：60 字以内，客观，不宣传。
- 适合人群：2-4 条。
- 报考提醒和风险：2-4 条。
- 检索关键词和检索轮次。
- 重复风险：正式库重复、知识库重复、同名/相近院校混淆。
- 建议动作：后续清洗 / 暂缓 / 跳过。

院校强候选建议至少 3 个来源，其中最好有：

- 学校官网。
- 本科招生网。
- 学院官网。
- 招生章程 / 招生计划 / 专业介绍。
- 研究生招生目录。
- 官方就业质量报告或教学质量报告。

## 4. 搜索深度要求

每个入选候选不能只搜一次。

企业至少做：

1. 主体名称 + 官网。
2. 主体名称 + 招聘 / 校园招聘 / 学校就业网。
3. 主体名称 + 方向词，例如电子雷管、爆破工程、含能材料、安全评价、防爆检测。

院校至少做：

1. 学校名称 + 招生网 / 本科招生。
2. 学校名称 + 学院 / 专业介绍。
3. 学校名称 + 研究生招生目录 + 方向词。

搜索后必须记录：

- 本轮广度搜索关键词。
- 单条候选核验关键词。
- 找到的有效来源。
- 仍缺什么来源。

## 5. 证据强弱口径

强：

- 官网、学校官网、招生网、学院官网、政府资料、上市公司公告、官方招聘公告、学校就业网原始招聘简章。

中：

- 权威媒体、集团新闻、招聘平台公司页、官方社媒历史内容、学校二级单位新闻、研究生招生网线索。

弱：

- 百科、工商聚合页、第三方院校库、招聘平台评论、论坛问答、自媒体、来源不明 PDF 镜像。

规则：

- 只有 1 个来源时，证据强弱最多写“中”，不能写“强”。
- 官网打不开或只有招聘平台/第三方转载时，写“中 / 待核对”。
- 弱来源不能单独支撑核心事实。
- 薪资、作息、分数线、招生人数、就业率没有强/中证据时统一写未知 / 待核对。

## 6. 文件产物要求

每轮知识库任务必须产出：

- 本轮 markdown 文件，保存到 `batches/` 对应子目录。
- 推荐配套 `.raw.json`，保存到 `raw-json/`。
- 更新 `INDEX.md`。

文件必须放在：

```text
E:\弹药网站\ammo-site\research-drafts\knowledge-pool\
```

推荐分层：

```text
batches/mixed/
batches/enterprise/
batches/university/
raw-json/
notes/
```

不允许只在聊天框里显示。

## 7. 和正式入库的区别

知识库阶段做到“入库预备级”，但不入库：

- 不运行 `npm run import:company`。
- 不运行 `npm run import:university`。
- 不改正式库。
- 不要求马上 dry-run。

后续正式入库时，再从知识库挑强候选，生成 clean JSON，跑 dry-run、评分和数据质量检查。
